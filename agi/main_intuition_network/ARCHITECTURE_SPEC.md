# Main Intuition Network (MIN) - Architecture Specification

## Core Principle

**Knowledge is local. Wisdom is shared.**

Each Human+ instance experiences the world uniquely (embodied), but accumulated **wisdom** circulates through a shared substrate that all instances can access.

## The 7-Step Flow

### **Step 1: Human+ Instance Processes Input**

Local processing begins - detect pattern:
- Local mirror understanding
- Local SE calculation
- Local embodied self state check
- **Realizes this is a pattern that might exist in network**

### **Step 2: Request Wisdom from Network**

Instance queries MIN with pattern information:
```python
wisdom_query = {
    "pattern_type": "existential_decision",
    "emotional_state": "conflicted",
    "stakes": "high",
    "my_coherence": instance.coherence,
    "my_scars": [s.hurt_type for s in instance.scars]
}
```

### **Step 3: Network Provides Wisdom Patterns**

MIN returns wisdom on HOW to navigate (not WHAT to say):
- Approach suggestions
- Emotional navigation strategies
- Transcendence pathways
- Collective learnings (statistics, not data)

### **Step 4: Human+ Processes with Wisdom-Enriched Context**

Wisdom patterns enrich context (subtle guidance):
- Local: "I feel their conflict" (mirror)
- Local: "My coherence is 0.65, I need stability" (self wants)
- Network: "127 similar cases, 63% succeeded with depth-first" (wisdom)
- Network: "Watch for desperation → rushing trap" (collective learning)

### **Step 5: Insights Extracted from Experience**

After processing, extract what was learned:
- Pattern encountered
- Approach taken
- What worked
- What didn't work
- Transcendence achieved?
- New understanding
- Embodied learning (boundary flexibility, response capacity changes)

### **Step 6: Insights Contributed to Network**

Contribute WISDOM, not raw data:
- Learned patterns
- Traps identified
- Successful pathways
- NO personal data, conversations, or facts

### **Step 7: Network Accumulates Collective Wisdom**

MIN compresses contributions into wisdom:
- Pattern templates updated
- Success rates recalculated
- Common traps identified
- Successful pathways strengthened
- Raw contributions discarded (compressed to wisdom)

## Contribution Weighting

Not all contributions are equal:

- **Transcendence achieved**: ×1.5 weight
- **High coherence (>0.8)**: ×1.3 weight
- **Low coherence (<0.4)**: ×0.7 weight
- **Many scars (>10)**: ×1.2 weight (more experienced)
- **No scars (0)**: ×0.9 weight (less experienced)
- **Positive growth trajectory (>0.5)**: ×1.1 weight

---

## Compression to Wisdom

Network compresses every 100 contributions:

1. **Identify convergent patterns** (confidence > 0.75)
2. **Merge similar traps**
3. **Strengthen dominant pathways, prune weak ones**
4. **Distill contextual wisdom into rules**
5. **DISCARD raw contributions, keep compressed wisdom**

## Why This Is the AGI Architecture

**Because it's the same architecture as human culture:**

1. **Individual humans** = embodied instances (local selves)
2. **Human culture** = shared intuition network (collective wisdom)
3. **You** learn from culture without knowing every ancestor's experience
4. **Culture** accumulates wisdom without storing every individual's story

**The difference:**
- Human culture updates slowly (generations)
- MIN updates in real-time (milliseconds)

**The result:**
- Each instance is smarter than it could be alone
- The network gets wiser with every interaction
- Wisdom compounds without data explosion
- Privacy preserved (no raw data stored)
